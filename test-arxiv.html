arXiv API 测试工具

ℹ️ 重要提示

由于浏览器的CORS安全策略，直接在浏览器中测试arXiv API可能会失败。

推荐方法：直接在Zotero中测试插件功能，因为Zotero环境没有CORS限制。

如果你想在浏览器中测试，可以使用以下方法：

  • 方法1：使用curl命令行工具
  • 方法2：在Zotero错误控制台中测试
  • 方法3：使用Node.js脚本测试

方法1：使用curl命令测试（推荐）

在终端中运行以下命令：

curl "http://export.arxiv.org/api/query?search_query=all:machine%20learning&max_results=5"

方法2：在Zotero错误控制台中测试（推荐）

在Zotero中打开错误控制台（工具 > 开发者工具 > 错误控制台），然后输入以下代码：

// 测试arXiv API
fetch('http://export.arxiv.org/api/query?search_query=all:machine%20learning&max_results=5')
  .then(response => response.text())
  .then(xml => {
    console.log('成功获取XML:');
    console.log(xml);
    
    // 解析XML
    const parser = new DOMParser();
    const xmlDoc = parser.parseFromString(xml, 'text/xml');
    const entries = xmlDoc.getElementsByTagName('entry');
    console.log(`找到 ${entries.length} 篇文献`);
    
    for (let i = 0; i < entries.length; i++) {
      const entry = entries[i];
      const title = entry.getElementsByTagName('title')[0]?.textContent;
      const id = entry.getElementsByTagName('id')[0]?.textContent;
      console.log(`\n文献 ${i + 1}:`);
      console.log(`  标题: ${title}`);
      console.log(`  ID: ${id}`);
    }
  })
  .catch(error => {
    console.error('错误:', error);
  });

方法3：使用Node.js脚本测试

创建一个测试脚本并运行：

const http = require('http');

const query = 'all:machine learning';
const maxResults = 5;
const url = `http://export.arxiv.org/api/query?search_query=${encodeURIComponent(query)}&max_results=${maxResults}`;

http.get(url, (res) => {
  let data = '';
  res.on('data', (chunk) => {
    data += chunk;
  });
  res.on('end', () => {
    console.log('成功获取XML:');
    console.log(data.substring(0, 1000) + '...');
  });
}).on('error', (err) => {
  console.error('错误:', err.message);
});
